# 安全政策

Crypto AI AutoTrading 涉及交易所 API、自动化执行和运行数据。请谨慎处理任何可能影响资金安全的问题。

## 支持范围

当前项目处于开源研究和工程实践阶段。默认支持最新 `main` 分支的安全修复。

## 报告安全问题

如果你发现以下问题，请不要公开提交包含可利用细节的 Issue：

- API key、secret、passphrase 或 webhook secret 泄露。
- 可绕过 `ALLOW_LIVE_ORDERS` 的真实下单路径。
- 可能导致非预期实盘交易、仓位放大或风控失效的问题。
- 远程访问、仪表盘暴露或命令执行相关风险。
- 日志、数据库、报告中泄露账户或交易敏感信息。

建议通过 GitHub 私有安全报告功能提交。如果仓库暂未开启该功能，请先提交一个不包含利用细节的 Issue，说明“需要私下披露安全问题”，维护者会提供后续联系方式。

## 凭据处理建议

- 不要提交 `.env`、数据库、日志、模型训练产物或交易记录。
- 交易所 API key 应使用最小权限，优先只读，实盘交易使用独立子账户。
- 开启交易所 IP 白名单、提现禁用和额度限制。
- 定期轮换 API key、LLM key 和 webhook secret。
- 一旦凭据被提交到 Git 或聊天记录中，应立即吊销并重新生成。

## 实盘保护原则

- 默认运行模式应为 `paper`。
- 真实下单必须同时满足 `RUNTIME_MODE=live` 和 `ALLOW_LIVE_ORDERS=true`。
- 任何贡献都不应默认打开真实下单能力。
- 任何执行路径都不应绕过滑点、仓位、熔断和 readiness 检查。

## 免责声明

安全修复不能消除交易本身的市场风险。使用者需要自行审计配置、权限、资金规模和运行环境。
